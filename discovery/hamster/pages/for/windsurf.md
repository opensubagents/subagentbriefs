The problem

## Windsurf is fast. But it needs direction.

### Sessions start without requirements

You open Windsurf with a rough idea. The AI generates code, you course-correct, and the result needs cleanup. Requirements were never in the loop.

### Context lives outside the editor

Product decisions are in docs, tickets, and Slack. Windsurf cannot access them. You summarize from memory, losing detail every time.

### No coordination across team sessions

Multiple developers using Windsurf on the same feature produce overlapping changes. No shared task boundaries means merge conflicts and rework.

## Hamster brings structure to Windsurf.

Write a brief in Hamster, and AI expands it into a full spec. Generate a plan with scoped tasks. The Hamster CLI syncs everything into .hamster/ — Windsurf reads the brief, reasoning, and team decisions directly.

[Explore the Hamster CLI→](https://tryhamster.com/product/studio/cli)

### Requirements before prompts

Hamster briefs define the outcome, scope, and constraints so Windsurf sessions stay on-target from the first prompt.

### Session-sized tasks

Hamster decomposes features into tasks that fit a single focused Windsurf session, preventing scope creep and wasted iterations.

### Always-on context via the CLI

The Hamster CLI syncs briefs, tasks, and plans into .hamster/ as markdown files in real time. Windsurf reads them on every action.

## How teams use Hamster with Windsurf

### Feature implementation

Before

Describe the feature in a prompt. Iterate as Windsurf misunderstands the scope. Manually fix the output.

After

The Hamster CLI syncs the full brief into your repo. Windsurf reads .hamster/ files and builds against the spec your team aligned on.

### Team coordination

Before

Two developers use Windsurf on the same feature. No shared task boundaries. Merge conflicts and duplicated work.

After

Hamster breaks the feature into non-overlapping tasks. Each developer picks a scoped task with clear acceptance criteria.

### Multi-session work

Before

Re-explain remaining work each session. Track progress manually.

After

The CLI keeps task status in sync. Each Windsurf session picks up where the last one left off with full context.
