# PRD to tasks in one command.

Drop in any requirements document — PRD, spec, Notion export — and Taskmaster generates a complete task list with dependencies, priorities, and implementation details.

Parsing

## One command to get started

Point Taskmaster at any text file and AI reads the document, understands the scope, and generates tasks with the right granularity. Use --research for Perplexity-backed context.

zsh — taskmaster

requirements.txt (PRD)

Build a user authentication system with email/password and OAuth...

Structure

## Smart dependencies built in

Generated tasks include logical dependencies, acceptance criteria, and implementation hints so you know what to work on first and when each task is done.

prd.txt

Generated Tasks12 tasks

Set up project structure

Implement core API

Build UI components

Customization

## Customize the output

Control task count, enable research-backed parsing, append to existing tasks, or force-replace them. Tailor the output to match your workflow.

`--num-tasks`Control how many top-level tasks to generate

`--research`Use Perplexity for research-backed parsing

`--append`Add to existing tasks instead of replacing

`--force`Overwrite existing tasks without prompting

Key capabilities

* Parse any text-based requirements document
* Generate tasks with logical dependencies
* Include implementation hints and acceptance criteria
* Configurable task count and granularity
* Research-backed parsing with Perplexity
* Append to or replace existing task lists
