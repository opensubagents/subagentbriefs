# Brief Status

Track where a [Brief](https://tryhamster.com/docs/hamster-studio/briefs) sits in its lifecycle so your team always knows what stage the work is at.

## Overview

Every [Brief](https://tryhamster.com/docs/hamster-studio/briefs) has a status that reflects its current stage, from initial writing through active [Delivery](https://tryhamster.com/docs/concepts/delivery) to completion. Status is visible on each [Brief](https://tryhamster.com/docs/hamster-studio/briefs) card in the list view and in the document header. You can update a [Brief](https://tryhamster.com/docs/hamster-studio/briefs)'s status at any time — the change takes effect immediately for everyone with access to the brief. Statuses are also used to filter and group [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) in the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list, making it easy to find work in a specific stage.

![Briefs list view showing status badges and filter controls for each stage](https://tryhamster.com/_site/images/features/briefs/brief-status/brief-status-overview-desktop-light.webp?v=6)![Briefs list view showing status badges and filter controls for each stage](https://tryhamster.com/_site/images/features/briefs/brief-status/brief-status-overview-desktop-dark.webp?v=6) 

## The Status Stages

[Briefs](https://tryhamster.com/docs/hamster-studio/briefs) move through the following stages:

| Status    | What it means                                                                                                                                                              |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Draft     | The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) is being written. Not yet ready for team review.                                                            |
| Refining  | The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) is being actively improved — typically during collaboration with the AI or after initial feedback.          |
| Aligned   | The team has agreed the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) is ready to proceed. Alignment votes from team members are visible in the Activity tab. |
| Shipping  | Work is actively in progress. [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) are being executed.                                                                |
| Delivered | The work has been completed and delivered.                                                                                                                                 |
| Done      | The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) and all associated work is fully closed out.                                                                |
| Archived  | The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) is no longer active and has been moved out of the default view.                                             |

## How It Works

1. See the current status — The status badge appears in the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) document header and on each [Brief](https://tryhamster.com/docs/hamster-studio/briefs) card in the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list.
2. Change the status — Open the brief. Click the status badge in the document header to open the status picker. Select a new status. The change saves immediately.
3. Filter by status — In the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list, click the stage filter button in the toolbar to show only [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) in one or more specific stages. You can select multiple stages at once.
4. Group by status — Use the display settings control in the toolbar and set "Group by" to Status. The list will organize all your [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) under their respective stage headings.
5. Archive a [Brief](https://tryhamster.com/docs/hamster-studio/briefs) — From the table view, click the three-dot menu on any [Brief](https://tryhamster.com/docs/hamster-studio/briefs) row and select "Archive". Archived [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) are hidden from the default view but remain accessible when you filter to show the Archived stage.

## Key Capabilities

* **Status on cards**: Every [Brief](https://tryhamster.com/docs/hamster-studio/briefs) card in the grid view shows the status as a colored badge so you can scan the list quickly.
* **Status filtering**: Filter the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list by one or multiple stages simultaneously. The filter applies instantly without a page reload.
* **Table view status**: In the table view, each row shows the [Brief](https://tryhamster.com/docs/hamster-studio/briefs)'s current status as a colored badge alongside the title, creator, and last updated time.
* **Alignment [Connection](https://tryhamster.com/docs/hamster-studio/connections)**: The Aligned status pairs with the Activity tab, where team members can cast alignment votes to indicate they're ready to proceed. See [Activity Timeline](https://tryhamster.com/docs/hamster-studio/briefs/activity-timeline) for details.

## Tips

* Move [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) through statuses as work progresses to give your team an at-a-glance view of where everything stands.
* Use the "Draft" status while you're still writing. Only move to "Refining" or "Aligned" once the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) is ready for others to review.
* Archive [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) that are no longer relevant rather than deleting them. Archived [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) retain all their content, context documents, [Tasks](https://tryhamster.com/docs/hamster-studio/tasks), and history.
* Group your [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list by Status when doing a weekly review to quickly see what's moving and what's stuck.
