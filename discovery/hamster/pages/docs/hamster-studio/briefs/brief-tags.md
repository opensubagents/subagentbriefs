# Brief Tags

Label [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) with tags so your team — and the AI — can find work by topic, theme, or any grouping that fits how you plan.

## Overview

Tags are short, freeform keywords you attach to briefs. They're shared across your workspace — once a tag exists, it appears in every team member's tag picker. You can assign multiple tags to a [Brief](https://tryhamster.com/docs/hamster-studio/briefs), filter the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list by tag, and ask the AI to pull up [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) by tag in chat.

Tags are designed to live where you already work. There's no separate tag management page — you create and assign tags directly in the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) detail header or from the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) table view, the same place you read and edit briefs.

## How It Works

1. Add a tag — Open a [Brief](https://tryhamster.com/docs/hamster-studio/briefs) or find it in the table view. Click the **+** button next to the existing tags. A compact dropdown appears showing tags your team has already used, with a search field to filter them. Select an existing tag or type a new name to create one immediately.
2. Tag from the list — In the table view, the tag pills are interactive. Add or remove tags inline without opening the brief.
3. Remove a tag — Click the **×** on any tag pill to remove it from the brief. The tag itself stays available for other briefs.
4. Filter by tag — In the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list toolbar, click the tag filter (shows "All tags" by default). Select one or more tags to narrow the list. [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) matching any of the selected tags are shown.
5. Filter from chat — Ask the AI in chat to "show me [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) tagged X" and the agent will return matching [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) as inline cards. The tag filter tool searches across your account.

## Key Capabilities

* **Inline tagging**: Add and remove tags from the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) header or from the table view. No separate tag manager.
* **Compact native dropdown**: The tag picker is a single inline combobox modelled on GitHub labels and Linear — fast to open, fast to filter, and fast to dismiss.
* **Shared tag vocabulary**: Tags are scoped to your workspace. Every tag any teammate creates shows up for everyone, so naming converges naturally over time.
* **Multi-tag filtering**: Select multiple tags in the filter dropdown to show [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) matching any of them. Your filter selection persists across sessions.
* **Filter by tag in chat**: Ask the AI to filter [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) by tag and get back a list of matching [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) as reference cards.
* **Instant tag creation**: Type a new tag name in the picker and it's created as soon as you select it. Names are normalised to lowercase and trimmed automatically. Names can be up to 50 characters.
* **Card and table display**: Tags appear as read-only badges on [Brief](https://tryhamster.com/docs/hamster-studio/briefs) cards in grid view and as interactive pills in table view. Sync badges (Linear, Jira, GitHub) are kept separate from tags so the two don't collide visually.
* **Tag limit**: A single [Brief](https://tryhamster.com/docs/hamster-studio/briefs) can have up to 20 tags. The add button is hidden when the limit is reached, and you'll get a clear error message if you somehow hit the cap from another surface.

## Tips

* Use tags for cross-cutting concerns that don't map to a single [Initiative](https://tryhamster.com/docs/hamster-studio/initiatives) — labelling [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) by team, sprint, priority theme, or customer segment.
* Keep tag names short and consistent. The picker shows existing tags from your workspace, so your team will naturally converge on a shared vocabulary over time.
* Combine tag filtering with status grouping for a focused view — filter to a specific tag and group by status to see what's in progress versus done.
* Ask the AI in chat to filter by tag when you want [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) surfaced as reference cards directly in the conversation, rather than navigating to the list.
