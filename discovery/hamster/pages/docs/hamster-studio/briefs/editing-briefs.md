# Editing Briefs

Write, refine, and deliver — all from one workspace, with the AI chat alongside the document and a deliver button that morphs into a PR badge once the work ships.

## Overview

The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) workspace pairs a rich text editor on one side with the AI chat panel on the other. As you write, you can ask the AI to expand a section, suggest requirements, or critique the scope — and it works with the actual content of your [Brief](https://tryhamster.com/docs/hamster-studio/briefs), not a summary. Multiple teammates can edit the same [Brief](https://tryhamster.com/docs/hamster-studio/briefs) at the same time, with cursors and changes appearing live.

Once the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) is sharp enough to act on, you generate a [Plan](https://tryhamster.com/docs/hamster-studio/plans), hit Deliver, and watch the deliver button track the work all the way to a pull request — without leaving the page.

## How It Works

1. Open a [Brief](https://tryhamster.com/docs/hamster-studio/briefs) — Click any [Brief](https://tryhamster.com/docs/hamster-studio/briefs) from the [Briefs](https://tryhamster.com/docs/hamster-studio/briefs) list. The AI chat opens on the left, the document on the right.
2. Write and edit — Click into the document and type. The editor supports headings, lists, bold, italic, links, images, and the rest of the standard rich text set.
3. Ask the AI for help — In the chat panel, ask for a rewrite, expansion, or critique. Mention `@hamster` to address the AI directly, or `@teammate` to bring a person into the thread.
4. Drop in context — Paste a Figma URL or upload a `.fig` file and the agent reads the design context. Drop a PDF or markdown file and it gets indexed for the AI to reference.
5. Switch tabs — The document panel has four tabs: **[Brief](https://tryhamster.com/docs/hamster-studio/briefs)** (the editor), **[Plan](https://tryhamster.com/docs/hamster-studio/plans)** (parent [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) and subtasks), **Context** (attached files), and **Activity** (alignment votes and history).
6. Generate a [Plan](https://tryhamster.com/docs/hamster-studio/plans) — From the [Plan](https://tryhamster.com/docs/hamster-studio/plans) tab, generate a [Plan](https://tryhamster.com/docs/hamster-studio/plans) from the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) body. Progress streams live in the chat as a card; the plan-generation card stays in sync even if you navigate away mid-flight.
7. Deliver — When the [Plan](https://tryhamster.com/docs/hamster-studio/plans) is ready, click Deliver. Pick a repo and branch, optionally toggle "Auto-open draft PR," and confirm. The deliver split-button keeps spinning while the work is in progress, then morphs into a PR-state badge — "Draft #42," "Merged #42" — that clicks straight through to GitHub.

## The Workspace Layout

The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) workspace is two panels:

* **Left panel — AI Chat**: The conversation thread for this brief. Use it to discuss, draft, and refine. Long replies and [Plan](https://tryhamster.com/docs/hamster-studio/plans) generation move into child threads automatically so the main thread stays readable.
* **Right panel — Document**: The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) content and its tabs ([Brief](https://tryhamster.com/docs/hamster-studio/briefs), [Plan](https://tryhamster.com/docs/hamster-studio/plans), Context, Activity).

Hide either panel using the column visibility controls. On mobile, the chat appears as a tab inside the document view.

## Key Capabilities

* **Real-time collaboration**: Any number of people can type in the same brief. Changes merge cleanly. Each person's cursor is colour-tagged with their name.
* **AI knows the document**: The chat reads your [Brief](https://tryhamster.com/docs/hamster-studio/briefs) automatically. You don't paste content into chat — the agent already has it.
* **Mentions**: Type `@` to mention a teammate or the Hamster AI. Mentioned teammates are added to the thread and notified.
* **Attachments grounded in the [Brief](https://tryhamster.com/docs/hamster-studio/briefs)**: Files you upload (`.fig`, PDF, markdown, images) and URLs you paste are extracted and indexed; the agent uses them when answering, drafting, or generating a plan. Pasted images survive the round trip and persist on the message.
* **AI image descriptions**: Images placed in the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) body are described in the background, and the description flows into chat answers, [Plan](https://tryhamster.com/docs/hamster-studio/plans) generation, and search.
* **Auto-generated description**: As you write, the [Brief](https://tryhamster.com/docs/hamster-studio/briefs)'s short description (the summary on the card in the list) is updated based on the content.
* **[Plan](https://tryhamster.com/docs/hamster-studio/plans) generation from chat or the [Plan](https://tryhamster.com/docs/hamster-studio/plans) tab**: When you ask the AI to [Plan](https://tryhamster.com/docs/hamster-studio/plans) from chat, the flow runs in its own child thread with a live progress card. Plan-gen and [Delivery](https://tryhamster.com/docs/concepts/delivery) cards land in whichever chat surface you triggered them from.
* **State-aware Deliver button**: One button covers the whole arc — "Deliver," "Delivering…," "Draft #42," "Merged #42" — and the click target morphs to match. No separate [Delivery](https://tryhamster.com/docs/concepts/delivery) tab; the state lives where you clicked it.
* **Opt-in draft PRs**: Tick "Auto-open draft PR" in the deliver dialog to have the PR opened automatically. Leave it off and the [Plan](https://tryhamster.com/docs/hamster-studio/plans) tab gives you a CTA that deep-links to a pre-filled GitHub compare page.
* **Reviewer read-only**: Teammates with the Reviewer role see the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) as fully read-only — no greyed-out controls, no cursor leaking outbound. They can still cast alignment votes and watch the document.
* **Recovers from interruptions**: If you navigate away during [Plan](https://tryhamster.com/docs/hamster-studio/plans) generation or [Delivery](https://tryhamster.com/docs/concepts/delivery), the cards still reach a terminal state when you come back. No more cards stuck on "in progress" forever.

## Tips

* Structure your [Brief](https://tryhamster.com/docs/hamster-studio/briefs) with headings. The AI uses document structure to understand sections and gives more precise responses when you ask about a specific part.
* Drop supporting material as context documents instead of pasting it into the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) body. The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) stays readable; the AI still has access to the details.
* Move the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) through statuses (Draft → Refining → Aligned → Shipping) as you progress so the team can see at a glance what's moving.
* Ask the agent in chat to update the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) status, log alignment, or pull a related [Brief](https://tryhamster.com/docs/hamster-studio/briefs) by tag. The agent has tools for all of those and will keep the document in sync.
