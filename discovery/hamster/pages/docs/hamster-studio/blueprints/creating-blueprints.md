# Creating Blueprints

Start documenting your company's context with a root [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints), then expand it into a structured hierarchy of child documents.

## Overview

A [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints) is a document your whole team shares. It can cover anything that grounds your work — product vision, target personas, strategic goals, brand voice, or team norms. You start with a single root document and grow it into as many child documents as you need. Each document you add becomes part of the same connected tree, and all of it feeds into the AI's understanding of your organization.

![Blueprints list view showing the document tree](https://tryhamster.com/_site/images/features/blueprints/creating-blueprints/blueprints-list-view-desktop-light.webp?v=6)![Blueprints list view showing the document tree](https://tryhamster.com/_site/images/features/blueprints/creating-blueprints/blueprints-list-view-desktop-dark.webp?v=6) 

## How It Works

1. Open the [Blueprints](https://tryhamster.com/docs/hamster-studio/blueprints) section — Select [Blueprints](https://tryhamster.com/docs/hamster-studio/blueprints) from the sidebar. If your workspace has no [Blueprints](https://tryhamster.com/docs/hamster-studio/blueprints) yet, a welcome screen explains the feature and offers a button to create your first one.
2. Create your first [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints) — Click "New [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints)" to open the creation dialog. Enter a title (required) and an optional description that summarizes what the document covers. Click Create.
3. Land in the editor — Hamster Studio opens the new document immediately. The editor is ready for you to start writing. The document title is editable inline at the top.
4. Add child documents — Once you have a root document, you can add sub-documents beneath it. Hover over any document in the left navigation panel and click the add button that appears, or use the "Add Child" option in the document's context menu. Child documents inherit the same tree and are visible in the navigation panel.
5. Keep building — There is no limit on depth or the number of documents. Build the structure that makes sense for your team — a flat list of top-level documents, or a nested hierarchy with sections and sub-sections.

## Key Capabilities

* **Title and description**: Every [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints) has a title (shown in navigation and page headers) and an optional description used as a summary when the AI references it.
* **Root and child documents**: Each workspace has one root-level [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints) entry point. All other documents in the tree are created as children of an existing document.
* **Inline title editing**: Click the document title in the editor to rename it. Changes save automatically and update the navigation tree in real time.
* **Immediate navigation**: After creating a [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints), you land directly in the editor. No extra steps needed.
* **Deletion**: You can delete any [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints) document from within the editor. If you delete a document that has a parent, you return to the parent. If you delete a root document, you return to the [Blueprints](https://tryhamster.com/docs/hamster-studio/blueprints) list.

## Tips

* Start broad. Your first [Blueprint](https://tryhamster.com/docs/hamster-studio/blueprints) might be a single document called "Company Overview" — you can always break it into sub-documents later as the content grows.
* Use the description field when creating a document. The AI uses descriptions to understand what each document is about, even before you have written the full content.
* You do not need to populate everything before [Blueprints](https://tryhamster.com/docs/hamster-studio/blueprints) become useful. Even a few paragraphs of product context will immediately improve [Brief](https://tryhamster.com/docs/hamster-studio/briefs) and [Plan](https://tryhamster.com/docs/hamster-studio/plans) quality.
