# Task Overview

[Tasks](https://tryhamster.com/docs/hamster-studio/tasks) are the individual units of work your team delivers — generated from your [Plan](https://tryhamster.com/docs/hamster-studio/plans), owned by a person, and tracked through to done.

## Overview

In Hamster Studio, [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) do not exist in isolation. They are the output of a [Plan](https://tryhamster.com/docs/hamster-studio/plans), which is itself generated from a brief. This means every [Task](https://tryhamster.com/docs/hamster-studio/tasks) carries context: it knows which [Brief](https://tryhamster.com/docs/hamster-studio/briefs) it belongs to, what the broader goal is, and where it sits in the [Delivery](https://tryhamster.com/docs/concepts/delivery) sequence. [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) represent discrete, actionable pieces of work — not vague to-do items, but scoped deliverables that a team member can pick up and drive to completion.

Each [Task](https://tryhamster.com/docs/hamster-studio/tasks) has a title, a description (which may include rich formatted content), a status, a priority level, an optional assignee, and an optional due date. [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) can also have subtasks nested beneath them when the work warrants further breakdown.

## How It Works

1. Start with a [Brief](https://tryhamster.com/docs/hamster-studio/briefs) — You create a [Brief](https://tryhamster.com/docs/hamster-studio/briefs) to capture the goal of a piece of work. The [Brief](https://tryhamster.com/docs/hamster-studio/briefs) contains objectives, context, and acceptance criteria. See [Creating Briefs](https://tryhamster.com/docs/hamster-studio/briefs/creating-briefs) for details.
2. Generate a [Plan](https://tryhamster.com/docs/hamster-studio/plans) — From the [Brief](https://tryhamster.com/docs/hamster-studio/briefs), Hamster Studio generates a [Plan](https://tryhamster.com/docs/hamster-studio/plans): an ordered sequence of [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) that describes how to deliver the work. Each step in the [Plan](https://tryhamster.com/docs/hamster-studio/plans) becomes a task. See [Generating Plans](https://tryhamster.com/docs/hamster-studio/plans/generating-plans) for how this works.
3. Work the [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) — Once [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) exist, they appear in the [Plan](https://tryhamster.com/docs/hamster-studio/plans) tab of the brief. You can view them as a flat list or grouped by status. Each [Task](https://tryhamster.com/docs/hamster-studio/tasks) can be opened to read its description, update its status, assign it to someone, and manage its subtasks.
4. Track progress — As [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) move from Todo to In Progress to Done, the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) shows an aggregate view of completion. When every [Task](https://tryhamster.com/docs/hamster-studio/tasks) is marked Done, the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) is effectively delivered.

## Key Capabilities

* **Three statuses**: [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) move through Todo, In Progress, and Done. You can change status from the [Task](https://tryhamster.com/docs/hamster-studio/tasks) list or from inside the [Task](https://tryhamster.com/docs/hamster-studio/tasks) detail view.
* **Four priority levels**: Low, Medium, High, and Urgent. Priority is set by the [Plan](https://tryhamster.com/docs/hamster-studio/plans) generator and can be updated manually.
* **Subtask nesting**: Every [Task](https://tryhamster.com/docs/hamster-studio/tasks) can contain subtasks. Subtasks follow the same status model and can themselves be further broken down. The [Task](https://tryhamster.com/docs/hamster-studio/tasks) row in the list shows a radial progress indicator when subtasks exist.
* **Complexity score**: The [Plan](https://tryhamster.com/docs/hamster-studio/plans) generator may attach a complexity rating (Low, Medium, High) to a task. This is displayed on the [Task](https://tryhamster.com/docs/hamster-studio/tasks) detail page and on [Task](https://tryhamster.com/docs/hamster-studio/tasks) row cards for quick scanning.
* **Dependencies**: [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) can have dependencies on other tasks. The count of dependency [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) is shown on the [Task](https://tryhamster.com/docs/hamster-studio/tasks) detail page.
* **Display ID**: Each [Task](https://tryhamster.com/docs/hamster-studio/tasks) gets a short identifier (e.g., `HAM-001`) that appears in breadcrumbs and [Task](https://tryhamster.com/docs/hamster-studio/tasks) rows. Click the copy button on any [Task](https://tryhamster.com/docs/hamster-studio/tasks) header to copy the ID to your clipboard.
* **Issue tracker sync**: When an issue tracker [Connection](https://tryhamster.com/docs/hamster-studio/connections) (like Linear) has two-way sync enabled, [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) sync as issues automatically. Status, priority, and assignee changes flow in both directions. See [Connections](https://tryhamster.com/docs/hamster-studio/connections) for available integrations.
* **Instant updates**: [Task](https://tryhamster.com/docs/hamster-studio/tasks) state — status, assignee, priority — updates live across all team members viewing the same [Brief](https://tryhamster.com/docs/hamster-studio/briefs) without reloading the page.

## The Task Lifecycle

A typical [Task](https://tryhamster.com/docs/hamster-studio/tasks) moves through the following states:

1. Todo — [Task](https://tryhamster.com/docs/hamster-studio/tasks) exists but no one has started it yet. This is the default state when a [Task](https://tryhamster.com/docs/hamster-studio/tasks) is first generated.
2. In Progress — Someone has picked up the [Task](https://tryhamster.com/docs/hamster-studio/tasks) or it has been assigned to an agent that has begun working on it.
3. Done — The work is complete. The [Task](https://tryhamster.com/docs/hamster-studio/tasks) is counted toward the [Brief](https://tryhamster.com/docs/hamster-studio/briefs)'s overall completion.

You can move [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) between states at any point. There is no enforced order — a [Task](https://tryhamster.com/docs/hamster-studio/tasks) can go back from Done to In Progress if rework is needed.

## Tips

* If a [Task](https://tryhamster.com/docs/hamster-studio/tasks) feels too large, use the AI expansion feature from inside the [Task](https://tryhamster.com/docs/hamster-studio/tasks) detail view to automatically generate subtasks. You can also add subtasks manually.
* The [Plan](https://tryhamster.com/docs/hamster-studio/plans) tab shows a "next best [Task](https://tryhamster.com/docs/hamster-studio/tasks)" indicator — a small dot on the [Task](https://tryhamster.com/docs/hamster-studio/tasks) most worth picking up next. This is determined by status: the first In Progress [Task](https://tryhamster.com/docs/hamster-studio/tasks) takes priority, followed by the first Todo task.
* [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) generated from the [Plan](https://tryhamster.com/docs/hamster-studio/plans) inherit context from the brief. The [Task](https://tryhamster.com/docs/hamster-studio/tasks) description will often contain enough information to act without referring back to the original brief.
