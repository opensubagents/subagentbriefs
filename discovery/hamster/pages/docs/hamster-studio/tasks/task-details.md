# Task Details

Open any [Task](https://tryhamster.com/docs/hamster-studio/tasks) to read its full description, review subtasks, check dependencies, and see all metadata in one place.

## Overview

Clicking a [Task](https://tryhamster.com/docs/hamster-studio/tasks) in the [Plan](https://tryhamster.com/docs/hamster-studio/plans) view opens the [Task](https://tryhamster.com/docs/hamster-studio/tasks) detail page. This is the primary workspace for a single piece of work. It shows the [Task](https://tryhamster.com/docs/hamster-studio/tasks)'s rich description, its subtasks, key metadata (status, priority, assignee, complexity, due date, dependencies), and, where applicable, the live status of any agent working on it. Changes made here — status updates, assignments, new subtasks — are reflected immediately in the [Plan](https://tryhamster.com/docs/hamster-studio/plans) list.

![Task detail page showing description, metadata fields, and subtasks section](https://tryhamster.com/_site/images/features/tasks/task-details/task-detail-view-desktop-light.webp?v=6)![Task detail page showing description, metadata fields, and subtasks section](https://tryhamster.com/_site/images/features/tasks/task-details/task-detail-view-desktop-dark.webp?v=6) 

## How It Works

1. Navigate to a [Task](https://tryhamster.com/docs/hamster-studio/tasks) — From the [Plan](https://tryhamster.com/docs/hamster-studio/plans) tab of any [Brief](https://tryhamster.com/docs/hamster-studio/briefs), click a [Task](https://tryhamster.com/docs/hamster-studio/tasks) row. The [Task](https://tryhamster.com/docs/hamster-studio/tasks) detail page opens with a breadcrumb trail linking back to the [Brief](https://tryhamster.com/docs/hamster-studio/briefs) and plan.
2. Read the description — The main body is an editable rich-text document. Content is generated from the [Plan](https://tryhamster.com/docs/hamster-studio/plans) and may include headings, bullet lists, code examples, and acceptance criteria. You can edit this content directly.
3. Review metadata — A metadata panel near the top shows Status, Assignee, Priority, Complexity, Created date, Due Date (if set), and Dependencies count. Each editable field opens a dropdown when clicked.
4. Work with subtasks — Below the description, a collapsible Subtasks section lists all child tasks. You can expand it to view progress, click any subtask to open its own detail page, manually add new subtasks, or use AI expansion to generate subtasks automatically.
5. Navigate via breadcrumbs — The breadcrumb trail shows: [Brief](https://tryhamster.com/docs/hamster-studio/briefs) title → [Plan](https://tryhamster.com/docs/hamster-studio/plans) → (parent [Task](https://tryhamster.com/docs/hamster-studio/tasks) if applicable) → current [Task](https://tryhamster.com/docs/hamster-studio/tasks) ID. Each segment is a link, so you can move back up the hierarchy without using the browser back button.

## Key Capabilities

* **Rich text description**: [Task](https://tryhamster.com/docs/hamster-studio/tasks) descriptions support headings, lists, code blocks, and other rich formatting. The content is editable directly — there is no separate edit mode to enter.
* **Live [Task](https://tryhamster.com/docs/hamster-studio/tasks) data**: The [Task](https://tryhamster.com/docs/hamster-studio/tasks) detail page subscribes to real-time updates. If another team member changes the status or assignee while you are viewing the page, your view updates automatically.
* **Subtask progress**: The Subtasks section shows how many subtasks are complete. Each subtask row shows its own status, assignee, and a link to its own detail page. Subtasks can be reordered by dragging.
* **AI subtask expansion**: When a [Task](https://tryhamster.com/docs/hamster-studio/tasks) has no subtasks, an "Expand Subtasks" button appears. Clicking it sends the [Task](https://tryhamster.com/docs/hamster-studio/tasks) to the AI, which generates subtasks based on the description and [Brief](https://tryhamster.com/docs/hamster-studio/briefs) context. The expansion runs asynchronously and subtasks stream in as they are created. This button is only available for top-level [Tasks](https://tryhamster.com/docs/hamster-studio/tasks), not subtasks.
* **Manual subtask creation**: An "Add Subtask" button opens an inline form. Type a title and press Enter to create the subtask immediately.
* **Dependency count**: The metadata grid shows how many other [Tasks](https://tryhamster.com/docs/hamster-studio/tasks) this [Task](https://tryhamster.com/docs/hamster-studio/tasks) depends on. Dependencies are set during [Plan](https://tryhamster.com/docs/hamster-studio/plans) generation and indicate ordering relationships.
* **Agent run status**: If the [Task](https://tryhamster.com/docs/hamster-studio/tasks) has been assigned to an AI agent, a status strip in the metadata area shows the current run status and progress percentage. If the agent creates a pull request, a link to it appears here.

## Tips

* The description is live-editable. Click anywhere in the content area and begin typing.
* Subtask nesting is recursive. A subtask can have its own subtasks, and clicking into one opens the same layout with its own Subtasks section.
* When a [Task](https://tryhamster.com/docs/hamster-studio/tasks) is deleted while you are viewing it, the page automatically redirects you back to the plan.
