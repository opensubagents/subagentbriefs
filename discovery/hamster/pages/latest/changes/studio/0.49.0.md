# Glimpse into any brief from a shared link

Shared brief links now show a preview page. Hamster now also revises plans mid-execution.

## Brief Sharing

People who open a shared [brief](https://tryhamster.com/product/studio/briefs) link now see a preview page. Team members skip the preview and land straight on the brief.

![The share page showing "Sign in to see this brief" with a brief preview card displaying the title, description, and Aligned status badge, and a Sign in to Hamster button](https://tryhamster.com/_site/images/releases/studio/0.49.0/share-page.webp)

Add teammates to a brief or invite someone new to Hamster from the brief header. [Learn more about sharing briefs](https://tryhamster.com/docs/hamster-studio/briefs/sharing-briefs).

![The Add to Brief dropdown showing a link copy button, search teammates field, and an Invite someone new to Hamster option](https://tryhamster.com/_site/images/releases/studio/0.49.0/share-dropdown.webp)

## Re-Planning

Hamster can now revise its [plan](https://tryhamster.com/product/studio/plans) mid-execution. If a task reveals an unexpected dependency, Hamster re-evaluates the remaining steps instead of continuing with the original plan.

## Improvements

* Hamster now reads the current brief when you ask it to revise, instead of working from memory
* Briefs now show their creator on the list view
* `hamster sync --brief` syncs a single brief instead of your entire workspace
* `hamster sync --team` selects a team without prompts, so scripts and CI pipelines can sync unattended
* Hamster now responds when @mentioned alongside other users in the same message
* You can now send messages with only an attachment and no text
* Pasting longer text into chat no longer converts it to an attachment unexpectedly
* [Blueprints](https://tryhamster.com/product/studio/blueprints) now show inline diffs when Hamster proposes changes, just like briefs

## Bug Fixes

* Fixed CLI auto-upgrade failing when checking for new versions
* Fixed MFA sessions losing authentication during long `watch` sessions
