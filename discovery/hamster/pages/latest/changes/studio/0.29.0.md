# Edit briefs together in real time

The new block-based editor supports real-time collaboration, change previews, and inline images.

## Block Editor

Briefs now use a block-based editor with slash commands, drag-to-reorder, and an inline toolbar. If you've used Notion, you'll feel at home.

Edit documents at the same time as teammates. You see their cursors and changes as they happen.

When Hamster suggests edits, they appear as highlighted diffs you can accept or reject before they're applied. [Learn more about real-time collaboration](https://tryhamster.com/docs/hamster-studio/collaboration/real-time-editing).

## Chat Threading

Start a side conversation from any message to explore a direction without cluttering the main thread. Long threads get automatic summaries.

You can now queue multiple messages while Hamster is still working. They're handled in order without losing context. [Learn more about thread messaging](https://tryhamster.com/docs/hamster-studio/conversations/thread-messaging).

## Notion and Jira Integrations

Connect your Notion workspace or Jira Cloud account so Hamster can use your pages and issues as context when working on briefs. Set up Notion or Jira from [Settings > Connections](https://tryhamster.com/docs/hamster-studio/connections/overview).

## Improvements

* Paste or drag images directly into briefs
* Export document content as Markdown
* @mention Hamster in chat to direct your question to it specifically
* Reply to any message in a thread, or post from a sub-thread back into the parent
* Share thread summaries with teammates via a link
* Hand off a task to Hamster and keep working on something else while it runs
