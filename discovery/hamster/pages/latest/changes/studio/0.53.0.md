# Skills that stay in sync across your whole team

Hamster CLI now syncs your skills in both directions. Initiatives now work directly from chat.

## Two-Way Skill Sync

Update a skill locally, run `hamster skills sync`, and your whole team gets the change. New skills created locally appear in Hamster on the next sync.

![The Hamster Studio Skills page with a terminal overlay running hamster skills sync, showing updated and already-synced skills](https://tryhamster.com/_site/images/releases/studio/0.53.0/cover.webp)

[Get started with skills](https://tryhamster.com/docs/hamster-studio/skills-methods/skills-overview).

## Initiatives in Chat

Hamster can now create initiatives, link briefs, reorder priorities, and check what's in flight directly from chat.

![A Hamster chat conversation creating the Platform Hardening initiative, linking three briefs, and updating its status to refining](https://tryhamster.com/_site/images/releases/studio/0.53.0/initiatives-chat.webp)

[Learn more about Initiatives](https://tryhamster.com/docs/hamster-studio/initiatives).

## Improvements

* Hamster now responds in whichever language you write in
* The Member role is now called Creator
* PDF text extraction no longer drops content from scanned and image-heavy documents

## Bug Fixes

* Fixed layout issues on the initiatives, briefs, and skills list views
* Fixed the creator filter dismiss button not appearing after page reload
* Fixed blueprint threads appearing empty on repeat visits
* Fixed an error page appearing after connecting Notion, even though the connection succeeded
